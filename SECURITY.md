# Security Policy

## Supported Versions

This repository is a template, so the latest `main` branch is the supported version.

## Reporting A Vulnerability

Please do not open a public issue for security-sensitive problems.

Instead:

1. contact the repository owner privately through GitHub
2. include reproduction steps, impact, and any suggested mitigation
3. avoid publishing proof-of-concept exploits until a fix is available

We will aim to acknowledge reports quickly and coordinate a responsible fix.

## Scope Notes

Because this is a template repository:

- secrets should never be committed
- placeholder configuration should stay non-sensitive
- downstream projects are responsible for their own runtime security posture
